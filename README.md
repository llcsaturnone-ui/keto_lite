# Простой трекер питания v7

Исправления критического бага:
- При массовом добавлении продуктов через JSON у нескольких продуктов мог получаться одинаковый id.
- Из-за одинакового id редактирование или удаление одного продукта могло ломать/затрагивать другие продукты.
- Теперь у каждого продукта всегда уникальный id.
- При загрузке старой базы приложение автоматически исправляет повторяющиеся id.
- Импорт backup теперь нормализует продукты и тоже чинит повторяющиеся id.
- Редактирование съеденной еды стало устойчивее, если старый foodId больше не найден.
- При переименовании продукта нельзя случайно создать дубль с уже существующим названием.

Формат для массового добавления:
[
  {"name":"Мандарин","calories":53,"protein":0.8,"fat":0.3,"carbs":13.3},
  {"name":"Киви","calories":61,"protein":1.1,"fat":0.5,"carbs":14.7}
]

Залей `index.html` вместо старого файла.
